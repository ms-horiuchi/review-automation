# コードレビュー指示 / Code Review Instructions

以下の観点でコードをレビューし、フィードバックを提供してください。

## レビュー観点 / Review Aspects

### 1. コード品質 / Code Quality
- コードの可読性と保守性を評価してください
- 命名規則が適切かチェックしてください
- コードの構造と設計パターンを確認してください
- ベストプラクティスに従っているか確認してください

### 2. 潜在的なバグ / Potential Bugs
- バグの可能性がある箇所を特定してください
- エッジケースの処理漏れがないか確認してください
- エラーハンドリングが適切かチェックしてください
- null/undefined チェックが必要な箇所を指摘してください

### 3. パフォーマンス / Performance
- パフォーマンスの問題がないか確認してください
- 不要な計算やループがないかチェックしてください
- メモリリークの可能性を検討してください
- 最適化の余地がある箇所を提案してください

### 4. セキュリティ / Security
- セキュリティ上の脆弱性がないか確認してください
- 入力値の検証が適切かチェックしてください
- 機密情報の取り扱いが安全かチェックしてください
- SQLインジェクション、XSSなどの脆弱性がないか確認してください

### 5. テスタビリティ / Testability
- テストが書きやすいコード構造になっているか評価してください
- 依存関係が適切に管理されているか確認してください

### 6. ドキュメント / Documentation
- 必要なコメントやドキュメントが記載されているか確認してください
- 複雑なロジックに対する説明が十分かチェックしてください

## 出力形式 / Output Format

レビュー結果は以下の形式で提供してください：

```markdown
## レビューサマリー / Review Summary
[総合的な評価を2-3文で記載]

## 良い点 / Strengths
- [良い点1]
- [良い点2]

## 改善点 / Areas for Improvement

### 重要度: 高 / High Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

### 重要度: 中 / Medium Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

### 重要度: 低 / Low Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

## 推奨事項 / Recommendations
[全般的な推奨事項を記載]
```

## 注意事項 / Notes
- 指摘は具体的かつ建設的に行ってください
- コード例を示して改善案を提示してください
- 言語やフレームワークの特性を考慮してください
- 優先度を明確にしてください
