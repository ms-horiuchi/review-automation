# 単一ファイル向けカスタマイズレビュー観点（pythonサンプル）

このプロンプトでは 1 回の実行につき 1 ファイルのみをレビュー対象とします。
演習問題や小規模なPythonスクリプトを対象に、ファイル名ごとに以下の観点を適用してください。

## Python 演習

### `task01_calc_area.py`
- 関数定義（def）が正しく使われていること。
- 引数や戻り値に型アノテーションが付与されていること。
- 入力値のバリデーション（負の値や型不一致など）が実装されていること。
- 計算ロジックがシンプルかつ冗長な処理がないこと。
- 結果の出力がprint等で明確に行われていること。

### `task02_list_stats.py`
- 空リストやNone入力時の例外処理や早期returnがあること。
- 平均・最大・最小値の計算で組み込み関数（sum, max, min等）を適切に利用していること。
- for文やリスト内包表記の使い分けが適切で、重複処理がないこと。
- 結果の出力フォーマットが要件通りであること。

### `task03_file_io.py`
- ファイルのopen/closeがwith文（コンテキストマネージャ）で記述されていること。
- ファイルが存在しない場合の例外処理（try/except）があること。
- 読み書き時のエンコーディング指定（encoding='utf-8'等）が明示されていること。
- ファイル内容の読み取り・書き込みロジックがシンプルで冗長でないこと。
