# 単一ファイル向けカスタマイズレビュー観点（サンプル）

このプロンプトでは 1 回の実行につき 1 ファイルのみをレビュー対象とします。
演習問題や小規模なシェルスクリプト（.sh）を対象に、ファイル名ごとに以下の観点を適用してください。

## Shell Script 演習

### `task01_print_date.sh`
- `date` コマンドの書式指定（+%Y-%m-%d等）が設問や指示文で指定された形式と一致していること。
- 変数展開時のクォート（"$VAR"）が適切に使われていること。
- 不要な一時ファイルやデバッグ出力が残っていないこと。

### `task02_sum_args.sh`
- 入力引数の個数チェック（$#）があり、エラー時はメッセージを出して終了していること。
- ループ処理（for arg in "$@"）で全引数を正しく合計していること。
- 数値以外の入力時にエラー処理があること。

### `task03_file_check.sh`
- ファイル存在チェック（[ -f "$filename" ]等）が正しく記述されていること。
- ファイル読み込み時のエラー処理やパーミッションチェックがあること。
- コマンド実行時の戻り値（$?）を適切に利用していること。
