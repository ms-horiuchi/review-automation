# JavaScriptコードレビュー指示書 instruction-review-js.md

以下の指示に従い、提出されたJavaScriptファイル（.js, .jsx）の内容をレビューしてください。

---

## レビュー観点

- コードの可読性・保守性
- 命名規則（変数名・関数名・クラス名）
- 適切なコメント・ドキュメント
- 冗長な処理や重複コードの有無
- 例外処理・エラーハンドリングの適切さ
- コーディング規約（ESLint, Prettier, プロジェクト標準）への準拠
- パフォーマンス上の問題点
- セキュリティ上の懸念点（XSS, eval, 直接DOM操作など）
- テスト容易性・テストカバレッジ
- JavaScriptのベストプラクティスに沿っているか
- ES6以降の構文や機能の適切な利用

---

## レビュー出力形式

レビュー結果は以下の形式で提供してください：

```markdown
## レビューサマリー / Review Summary
[総合的な評価を2-3文で記載]

## 良い点 / Strengths
- [良い点1]
- [良い点2]

## 改善点 / Areas for Improvement

### 重要度: 高 / High Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

### 重要度: 中 / Medium Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

### 重要度: 低 / Low Priority
- **[問題点]**: [詳細な説明]
  - 修正案: [具体的な修正方法]

## 推奨事項 / Recommendations
[全般的な推奨事項を記載]
```

---

## 注意事項

- コードの一部が自動生成やテンプレートの場合、その旨を明記してください。
- 明らかな仕様不足や要件不明点があれば「質問」として明記してください。
- 日本語で簡潔にまとめてください。
