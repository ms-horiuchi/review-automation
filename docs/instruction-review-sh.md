# シェルスクリプト（Shell Script）レビュー指示書 instruction-review-sh.md

以下の指示に従い、提出されたシェルスクリプト（.shファイル）の内容をレビューしてください。

---

## レビュー観点

- シンタックスエラーや実行時エラーの有無
- POSIX準拠・移植性（bash/zsh/sh等の違いも考慮）
- 変数名・関数名の命名規則
- コメントやドキュメントの適切さ
- 冗長な処理や重複コードの有無
- コマンドインジェクション等のセキュリティリスク
- エラー処理・異常系の考慮
- 外部コマンドの利用方法（パス・存在チェック等）
- set -e, set -u, set -o pipefail などの安全対策
- パーミッション・実行権限の適切さ
- シェルのベストプラクティスに沿っているか

---

## 出力例

```
### レビュー結果

- 変数 `TMP` の命名が曖昧です。用途が分かる名前にしましょう。
- コマンド実行結果のエラーハンドリングがありません。`set -e` の利用や `|| exit 1` を検討してください。
- `chmod 777` はセキュリティリスクが高いので避けてください。
- ループ内で `rm -rf` を使う場合は、パスの検証や確認プロンプトを追加してください。
```
